## 🟡 问题二：Agent 架构缺陷

### 缺陷1：串行无反馈，信息孤岛

```
现在：行情Agent → 舆情Agent → 风控Agent → 报告Agent
             ↑                               ↓
         无法根据报告质量要求补充数据       报告写完就结束
```

```python
# 改进：报告Agent发现信息缺失时，能触发补充查询
# backend/graph.py

def run_report_generation(state: FundAnalysisState) -> FundAnalysisState:
    
    # 检查关键信息是否缺失
    missing_info = []
    if "mock" in state.get("market_analysis", ""):
        missing_info.append("业绩数据为模拟数据")
    if not state.get("fund_name") or state["fund_name"] == state["fund_code"]:
        missing_info.append("基金名称未确认")
    
    # 在 prompt 里明确告知缺失情况
    synthesis_query = f"""
    ...
    
    ⚠️ 数据质量说明（必须在报告中如实注明）：
    {chr(10).join(f'- {m}' for m in missing_info) if missing_info else '- 数据完整'}
    
    要求：对于使用模拟数据的指标，报告中必须标注「数据待核实」
    """
```

```python
# backend/graph.py - 在行情分析后加一个验证节点

def validate_data_quality(state: FundAnalysisState) -> FundAnalysisState:
    """
    检查数据质量，标记哪些是真实数据，哪些是mock
    """
    market = state.get("market_analysis", "")
    
    data_quality = {
        "has_real_performance": "akshare" in market and "mock" not in market,
        "has_manager_info": "mock" not in state.get("market_analysis",""),
        "data_completeness": "高" if "mock" not in market else "低（含模拟数据）"
    }
    
    return {**state, "data_quality": data_quality}
```
